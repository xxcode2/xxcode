GenSynAI Testnet Installation Guide

Complete guide to install, configure and run GenSynAI on the testnet environment

System Requirements

CPU

4+ cores, 2.5+ GHz

RAM

16GB+ (32GB recommended)

Storage

100GB+ SSD (NVMe preferred)

Network

100+ Mbps, stable connection

OS

Ubuntu 20.04+ or similar Linux

Wallet

Testnet tokens for transactions

Installation Steps

1

Update your system

First, make sure your system is up to date:

sudo apt update && sudo apt upgrade -y
Copied!
2

Download the installation script

wget -O gensynai_install.sh https://raw.githubusercontent.com/xxcode2/xxcode/main/scripts/gensynai_testnet_install.sh
Copied!
3

Make the script executable

chmod +x gensynai_install.sh
Copied!
4

Run the installation script

./gensynai_install.sh
Copied!

The script will prompt you for your wallet address and other configuration options. Follow the on-screen instructions.

5

Verify the installation

gensynai --version
Copied!

If you see the version number, GenSynAI has been successfully installed!

1

Install dependencies

sudo apt update && sudo apt install -y build-essential git curl wget jq make gcc g++ libssl-dev pkg-config
Copied!
2

Install Rust

curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
Copied!

Follow the on-screen instructions and select option 1 for the default installation. Then, update your PATH:

source $HOME/.cargo/env
Copied!
3

Clone the repository

git clone https://github.com/xxcode2/gensynai.git
cd gensynai
Copied!
4

Switch to testnet branch

git checkout testnet
Copied!
5

Build from source

cargo build --release
Copied!

This process may take 15-30 minutes depending on your system specifications.

6

Create configuration file

Create a config directory and configuration file:

mkdir -p ~/.gensynai
nano ~/.gensynai/config.toml
Copied!

Add the following configuration (replace with your wallet address):

[node]
network = "testnet"
endpoint = "https://testnet-rpc.gensynai.io"
wallet_address = "YOUR_WALLET_ADDRESS"
log_level = "info"

[validator]
stake_amount = 1000
commission_rate = 0.05

[storage]
path = "~/.gensynai/data"
max_size_gb = 50
Copied!

Save the file by pressing Ctrl+X, then Y, then Enter.

7

Start the GenSynAI node

./target/release/gensynai --testnet
Copied!

Your GenSynAI node should now be running on the testnet!

Install Docker

If you don't have Docker installed:

curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

mkdir -p ~/.gensynai

Create configuration file

nano ~/.gensynai/config.toml

Add the following configuration (replace with your wallet address):

[node]

4

Run the Docker container

docker run -d \
  --name gensynai-testnet \
  -v ~/.gensynai:/config \
  -v ~/.gensynai/data:/data \
  -p 8545:8545 \
  -p 30303:30303 \
  --restart unless-stopped \
  xxcode2/gensynai:testnet
Copied!

Your GenSynAI node should now be running in a Docker container!

5

Check container logs

docker logs -f gensynai-testnet
Copied!

Configuration Options

Option Description Default
network Network to connect to (testnet or mainnet) testnet
endpoint RPC endpoint for the network https://testnet-rpc.gensynai.io
wallet_address Your wallet address for rewards Required
log_level Logging level (debug, info, warn, error) info
stake_amount Amount of tokens to stake as validator 1000
commission_rate Validator commission rate (0.01-0.20) 0.05
storage.path Path to store blockchain data ~/.gensynai/data
storage.max_size_gb Maximum storage size in GB 50

Node Management

Running as a Systemd Service

For better reliability, you can run GenSynAI as a systemd service:

1. Create a systemd service file:

sudo nano /etc/systemd/system/gensynai.service
Copied!

2. Add the following content:

[Unit]
Description=GenSynAI Testnet Node
After=network.target

[Service]
User=$USER
Type=simple
ExecStart=/home/$USER/gensynai/target/release/gensynai --testnet
Restart=on-failure
RestartSec=10
LimitNOFILE=65535

[Install]
WantedBy=multi-user.target
Copied!

3. Enable and start the service:

sudo systemctl daemon-reload
sudo systemctl enable gensynai
sudo systemctl start gensynai
Copied!

4. Check the service status:

sudo systemctl status gensynai
Copied!

Monitoring Your Node

Check the node status and logs:

View logs:

sudo journalctl -u gensynai -f
Copied!

Check node status via API:

curl -X GET http://localhost:8545/api/v1/status
Copied!

Updating Your Node

To update your GenSynAI node to the latest version:

cd ~/gensynai
git pull
git checkout testnet
cargo build --release
sudo systemctl restart gensynai
Copied!

Troubleshooting

Connection Issues

If your node cannot connect to the testnet:

Error: Failed to connect to testnet RPC endpoint

Solution:

Check your internet connection and firewall settings. Make sure ports 8545 and 30303 are open.

sudo ufw allow 8545
sudo ufw allow 30303

Syncing Issues

If your node is stuck syncing:

Warning: Node is not syncing or sync is very slow

Solution:

Try restarting the service and check your system resources:

sudo systemctl restart gensynai
htop

Validator Not Receiving Rewards

If your validator is not receiving expected rewards:

Warning: No rewards received in the last 24 hours

Solution:

Check if your validator is properly staked and active:

curl -X GET http://localhost:8545/api/v1/validator/status

Ensure your wallet address is correctly configured in the config file.
